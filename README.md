# NeuroMule

English version: [README.en.md](README.en.md)

Telegram-бот с текстовым чатом (OpenRouter), генерацией изображений/видео/музыки, тарифами, промокодами и проверкой подписки на канал.

## Требования

- Python **3.11+** (в Docker используется образ `python:3.11-slim`)
- Аккаунт и токен бота у [@BotFather](https://t.me/BotFather)
- Ключ [OpenRouter](https://openrouter.ai/) для работы нейросетевых функций

## Быстрый старт

1. Клонировать репозиторий и перейти в каталог проекта.

2. Создать виртуальное окружение и установить зависимости:

   ```bash
   python -m venv venv
   ```

   Windows:

   ```text
   venv\Scripts\activate
   ```

   Linux / macOS:

   ```text
   source venv/bin/activate
   ```

   ```bash
   pip install -r requirements.txt
   ```

3. Скопировать шаблон переменных окружения и заполнить секреты:

   ```bash
   copy .env.example .env
   ```

   На Linux/macOS: `cp .env.example .env`

   Обязательно задайте минимум:

   - `TG_TOKEN` — токен бота от BotFather  
   - `OPENROUTER_API_KEY` — ключ OpenRouter  
   - `CHANNEL_ID` и `CHANNEL_URL` — ваш канал для проверки подписки  
   - `ADMIN_IDS` — ваш числовой Telegram user id (через запятую, если несколько админов)

   Полный список переменных см. в файле `.env.example`.

4. Запуск Telegram-бота (режим по умолчанию):

   ```bash
   python main.py
   ```

   Переменная `NEUROMULE_PLATFORM` в `.env` может быть `telegram`, `vk`, `max` или `api` / `miniapp` / `fastapi` — см. комментарии в `main.py`.

### Доступ к Telegram API

Если при старте ошибки вида «cannot connect to api.telegram.org», проверьте сеть, файрвол или VPN: до серверов Telegram должен быть доступ по HTTPS (порт 443).

### Один экземпляр на машине

Второй запуск `python main.py` на том же компьютере завершится, если порт блокировки занят (`NEUROMULE_TELEGRAM_LOCK_PORT`, по умолчанию `45678`). Закройте лишний процесс или смените порт в `.env`.

## Запуск в Docker

В каталоге с проектом должен лежать заполненный `.env` (в образ токены не вшиваются).

```bash
docker compose up -d --build
```

База и логи монтируются в тома (см. `docker-compose.yml`). В контейнере `DB_PATH` указывает на `/app/data/neuromule_base.db`.

### Деплой на VPS (Timeweb и аналоги)

Подходит **VPS с Docker** (на общем хостинге без SSH/Docker бот обычно не запускают как долгоживущий процесс).

1. На сервере: установите Git и Docker Engine + Compose plugin (или следуйте инструкции хостинга).
2. Залейте код: клонирование репозитория (`git clone …`) или загрузка каталога по **SFTP/SCP**.
3. В каталоге проекта создайте **`.env`** на сервере (скопируйте с локальной машины или соберите из `.env.example`). Файл **не** должен попадать в Git.
4. Запуск:

   ```bash
   docker compose up -d --build
   ```

5. Логи контейнера: `docker compose logs -f`

Опционально тома на диске хоста: в `.env` задайте `NM_DATA_VOL` и `NM_LOGS_VOL` (см. комментарии в `docker-compose.yml`).

Убедитесь, что с **сервера** открывается `api.telegram.org:443` (на части VPS за пределами РФ это проще, чем с домашнего интернета без VPN).

## Админка (в Telegram)

Для user id из `ADMIN_IDS` в чате с ботом:

- `/admin` — панель: статистика, рассылка  
- `/give_energy <user_id> <amount>` — начислить энергию  
- `/add_promo <код> <награда> <число_активаций>` — создать промокод  

Опционально `ADMIN_USERNAME` (без `@`) — кнопка «Администратор» в разделе поддержки (личная ссылка), права админки от этого не зависят.

## Структура проекта (кратко)

| Путь | Назначение |
|------|------------|
| `main.py` | Точка входа, выбор платформы (`NEUROMULE_PLATFORM`) |
| `platforms/` | Интерфейсы мессенджеров (Telegram и др.) |
| `services/` | Бизнес-логика, БД, интеграции |
| `content/` | Тексты и подписи для пользователя |
| `api/` | FastAPI для Mini App при режиме `api` |
| `tests/` | Автотесты |

## Безопасность

- Не коммить файл `.env` и не публиковать токены.  
- В репозитории держите только `.env.example` без реальных ключей.
